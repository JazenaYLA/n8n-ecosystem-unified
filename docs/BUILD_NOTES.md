## Pre-Flight Checklist

Before running `setup.sh`, have these ready:

- [ ] Anthropic API key (`claude-sonnet-4.x` model access required)
- [ ] Telegram Bot token (from `@BotFather`) + your personal Telegram Chat ID
- [ ] OpenAI API key OR Voyage API key (for embeddings — default: `text-embedding-3-small`)
- [ ] A domain name pointed at your server (optional but needed for HTTPS credential form)
- [ ] Brave Search API key (optional — for MCP Builder doc search)
- [ ] Docker + `psql` client installed on host **before** running setup
- [ ] Minimum ~8GB RAM recommended (Crawl4AI alone reserves 4GB)

## Critical Gotchas

### 1. Credential Names Are Hardcoded — Do NOT Rename

Workflows match credentials **by exact name string**. Renaming them in the n8n UI causes silent failures across all dependent tools.

| Credential | Exact Name Required |
|---|---|
| Anthropic | `Anthropic API` |
| Telegram | `Telegram Bot` |
| PostgreSQL | `Supabase Postgres` |

### 2. Postgres Credential Cannot Be Created via API

The n8n REST API does not reliably create Postgres credentials. `setup.sh` attempts a CLI fallback (`n8n import:credentials`), but if that fails you **must** create it manually in the n8n UI under Settings → Credentials.

**Fields to use:**
- Host: `db` (Docker service name — NOT `localhost`)
- Port: `5432`
- Database: `postgres`
- User: `postgres`
- Password: value from `POSTGRES_PASSWORD` in your `.env`

### 3. Webhook Registration Bug — Deactivate/Reactivate Required

After `setup.sh` activates workflows via the n8n API, webhooks may **not register**. Symptom: Telegram messages are received but agent never responds.

**Fix:** In the n8n UI, open the main agent workflow → toggle OFF → toggle ON. Forces n8n to re-register the webhook. No automated workaround exists.

### 4. `specifyInputSchema` Is Silently Ignored on Workflow Create

If you build MCP tool workflows using `toolCode` nodes with `specifyInputSchema: true`, this field is **silently ignored** when imported via the n8n API. Parameters won't bind.

**Correct pattern:** Always use `toolWorkflow` → sub-workflow for parametrized tools. Parameters arrive via `$json.param` in the sub-workflow, which always works.

### 5. N8N_ENCRYPTION_KEY Must NOT Change After First Run

`N8N_ENCRYPTION_KEY` is auto-generated by `setup.sh`. All stored credentials are encrypted with this key. Regenerating it — or re-running `setup.sh` on an existing install — makes **every credential permanently unreadable**.

```bash
# Back up your key immediately after first run
grep N8N_ENCRYPTION_KEY .env >> ~/.n8n-claw-secrets.backup
```

### 6. Docker Host IP for Internal n8n Communication

Workflows calling the n8n API from inside Docker use `N8N_INTERNAL_URL=http://172.17.0.1:5678`.

- `localhost` inside a container = the container itself, NOT the host
- `172.17.0.1` = Docker bridge gateway (host IP as seen from containers)
- Supabase/PostgREST always uses Docker DNS: `http://kong:8000`

### 7. Kong Config Is Generated — Never Edit `kong.yml` Directly

`supabase/kong.yml` is a template. `setup.sh` generates `kong.deployed.yml` which is the actual mounted file. Editing `kong.yml` has no effect on a running instance.

To change Kong config: edit `setup.sh`'s generation block and re-run, or directly modify `supabase/kong.deployed.yml` and restart the Kong container.

### 8. Agent Behavior Lives in the Database, Not Workflow Code

The system prompt is built at runtime by querying `soul` and `agents` tables via PostgREST. Editing workflow JSON **does not change agent personality or behavior**.

```bash
# Inspect what's driving agent behavior
PGPASSWORD=yourpw psql -h localhost -U postgres -d postgres \
  -c "SELECT key, LEFT(content,120) FROM soul;"
```

### 9. jsDelivr CDN Caches `@master` for Hours

The MCP skill library fetches from jsDelivr CDN with `@master` tag. Adding a new skill and trying to install it immediately returns the cached old manifest.

**Fix:** Pin `CDN_BASE` to a specific commit hash:
```
https://cdn.jsdelivr.net/gh/freddy-schuetz/n8n-claw-templates@<commit-sha>/templates/
```

Purge cache manually:
```bash
curl https://purge.jsdelivr.net/gh/freddy-schuetz/n8n-claw-templates@<hash>/templates/index.json
```

### 10. Workflow IDs Are Not Stable Across Installs

Every fresh n8n import generates new IDs. `setup.sh` patches `REPLACE_*` placeholders post-import, but manually re-importing a workflow breaks all agent tool references until re-patched.

Tracked placeholders auto-patched by `setup.sh`:
- `REPLACE_REMINDER_FACTORY_ID`
- `REPLACE_WORKFLOW_BUILDER_ID`
- `REPLACE_MCP_BUILDER_ID`
- `REPLACE_LIBRARY_MANAGER_ID`
- `REPLACE_PROJECT_MANAGER_ID`
- `REPLACE_SUB_AGENT_RUNNER_ID`
- `REPLACE_AGENT_LIBRARY_MANAGER_ID`

## Docker / Container Notes

### Service Start Order

`n8n` waits for `db` health check (20 retries × 5s = up to 100s). On slow storage (e.g., HDD-backed Proxmox LXC), PostgreSQL first-run migration can exceed this. Increase `retries` in `docker-compose.yml` if n8n crashes on first startup.

### Crawl4AI Memory Requirements

Runs headless Chromium. Hard-limited to 4GB RAM + 2GB shared memory in the default compose file. On homelab nodes with < 8GB total RAM, reduce:

```yaml
shm_size: "1g"
deploy:
  resources:
    limits:
      memory: 2G
```

### PostgreSQL and Supabase Studio Are Localhost-Only

Both `5432` and `3001` are bound to `127.0.0.1`. To connect remotely via DBeaver, TablePlus, or browser:

```bash
ssh -L 5432:localhost:5432 -L 3001:localhost:3001 user@your-server
# Then connect to localhost:5432 / http://localhost:3001
```

## Embedding / Vector Memory Notes

### Dimension Mismatch Silently Breaks Memory

`memory_long` table has a fixed vector dimension (default: **1536** for OpenAI `text-embedding-3-small`). Switching providers without updating the schema causes silent failures — no errors, memory just stops working.

| Provider | Model | Dimensions |
|---|---|---|
| OpenAI (default) | `text-embedding-3-small` | 1536 |
| OpenAI | `text-embedding-3-large` | 3072 |
| Voyage | `voyage-3-lite` | 512 |
| Ollama | `nomic-embed-text` | 768 |

To change providers after initial setup: drop/recreate `memory_long` with the new dimension, then update `tools_config` in the DB.

### Embeddings Config Is Read from DB, Not `.env` at Runtime

`.env` embedding vars are only used by `setup.sh` to seed `tools_config`. After initial setup, change providers by updating the DB directly:

```sql
UPDATE tools_config SET value = 'voyage' WHERE key = 'embedding_provider';
UPDATE tools_config SET value = 'your-key' WHERE key = 'embedding_api_key';
```

## Separate Repos Embedded Inside n8n-claw (Important for Forks)

Two Git repos are embedded inside `n8n-claw/` but are **NOT git submodules** — they're independent repos:

| Path | Repo | Branch |
|---|---|---|
| `n8n-claw-agents/` | `freddy-schuetz/n8n-claw-agents` | `master` |
| `../n8n-claw-templates/` | `freddy-schuetz/n8n-claw-templates` | `master` |

- `git clone` of n8n-claw does **NOT** clone these automatically
- Always `cd` into each and `git push` separately — never via the parent repo
- When forking n8n-claw, fork these two repos separately and update CDN base URLs in library manager workflows

## Security Notes (Homelab / Self-Hosted)

> ⚠️ n8n had two critical RCE CVEs in late 2025/early 2026 (CVE-2025-68613 and CVE-2026-25049, CVSS 9.4). Any authenticated user who can create/edit workflows could own the host server. Ensure you are running a patched version.

- n8n port `5678` is exposed on `0.0.0.0` — put it behind Caddy/Traefik with auth if network-accessible
- `N8N_SECURE_COOKIE=false` in compose is for HTTP-only installs — set `true` when using HTTPS
- Rotate `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_KEY`, and `SUPABASE_JWT_SECRET` if PostgREST is exposed beyond localhost
- Never expose the n8n API key in workflows or public repos

## Useful Debug Commands

```bash
# Tail all service logs
docker logs -f n8n-claw
docker logs -f n8n-claw-db
docker logs -f n8n-claw-kong
docker logs -f n8n-claw-crawl4ai

# Inspect agent system prompt state
PGPASSWORD=yourpw psql -h localhost -U postgres -d postgres -c \
  "SELECT key, LEFT(content,200) FROM soul UNION ALL SELECT key, LEFT(content,200) FROM agents ORDER BY key;"

# Reimport a single workflow without full reinstall
N8N_KEY=your_api_key
curl -s -X POST "http://localhost:5678/api/v1/workflows" \
  -H "X-N8N-API-KEY: $N8N_KEY" \
  -H "Content-Type: application/json" \
  -d @workflows/deployed/mcp-builder.json

# Apply schema changes only
PGPASSWORD=yourpw psql -h localhost -U postgres -d postgres \
  -f supabase/migrations/001_schema.sql
```
